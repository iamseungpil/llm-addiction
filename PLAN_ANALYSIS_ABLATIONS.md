# V10 기존 분석 Ablation 계획서

> **목표**: V10의 분석 파이프라인에서 임의로 선택된 하이퍼파라미터와 설계 결정을 변화시켜, 결론이 얼마나 견고한지 검증한다.
>
> **한 줄 요약**: "PCA 50이 아니라 100이었으면? LogReg가 아니라 MLP였으면? 결론이 바뀌는가?"

모두 **기존 데이터 재분석**으로 가능하며, 새 게임 실행이 필요 없다.

---

## 왜 필요한가

V10의 분석 파이프라인은 여러 설계 결정을 포함한다:

```
StandardScaler → PCA(50) → LogReg(C=1.0) → 5-fold → 순열 200회
```

이 중 **PCA 차원(50)**, **분류기(LogReg)**, **순열 횟수(200)** 등은 근거 없이 선택되었다. 이 선택을 바꿨을 때 결론이 유지되면 견고한 결과이고, 바뀌면 해당 선택에 의존하는 취약한 결과이다.

---

## Ablation 목록 (우선순위순)

### A1: PCA 차원 민감도

**소요: 2-3시간**

| 검증 대상 | V10 선택 | 근거 |
|---------|---------|------|
| PCA 성분 수 | 50 | 미제시 |

**방법**: PCA 차원을 {10, 20, 50, 100, 200, 전체(PCA 없음)}로 변화시키며 BK 분류 AUC를 측정한다. 설명된 분산 비율(explained variance ratio)도 함께 보고한다.

**판단 기준**:
- 50 전후로 AUC가 포화 → 현재 선택 정당화
- 100 이상에서 AUC가 유의하게 상승 → 정보 손실 발생, 보고서 결과가 과소 추정
- PCA 없이도 AUC 유사 → PCA 불필요

---

### A2: 순열 검정 횟수 확대

**소요: 반나절 (계산 시간)**

| 검증 대상 | V10 선택 | 학술 표준 |
|---------|---------|---------|
| 순열 횟수 | 200 | 1,000+ |

**방법**: 핵심 분석(교차 도메인 전이, F1 교차 배팅 전이)에 대해 순열 검정을 1,000회로 확대한다.

**판단 기준**:
- p-value가 여전히 < 0.01 → 현재 결론 유지, 정밀도만 개선
- 일부 분석에서 p > 0.05 → 해당 결과의 유의성 재평가 필요

현재 "p = 0.000"으로 보고된 값들의 실제 정밀도가 확보된다.

---

### A3: 비선형 분류기 비교

**소요: 3-4시간**

| 검증 대상 | V10 선택 | 대안 |
|---------|---------|------|
| 분류기 | LogReg (선형) | MLP, SVM-RBF |

**방법**: 동일 파이프라인에서 분류기만 교체하여 AUC를 비교한다.

| 분류기 | 특징 |
|--------|------|
| LogReg (현재) | 선형 분리만 포착 |
| MLP (2-layer, 64-32) | 비선형 패턴 포착 |
| SVM-RBF | 비선형, 커널 기반 |

**판단 기준**:
- 비선형 AUC ≈ LogReg AUC → "BK가 선형 분리 가능하다"는 주장 강화
- 비선형 AUC >> LogReg AUC → 비선형 BK 패턴 존재, 현재 과소 추정

---

### A4: 층별 전체 분석 (LLaMA)

**소요: 반나절**

| 검증 대상 | V10 선택 | 문제 |
|---------|---------|------|
| LLaMA 분석 층 | L8, L12, L22, L25, L30 (5개) | 32개 중 5개만. 선택 근거 불명 |

Gemma는 42개 층 전체를 분석했으나 LLaMA는 5개만 분석했다. 체크포인트 사이의 층 특이적 현상이 누락될 수 있다.

**방법**: LLaMA의 나머지 27개 층에 대해서도 BK 분류 AUC를 측정한다. 최소한 핵심 분석(BK 분류, 보편적 BK 뉴런 수)만이라도.

**판단 기준**:
- 기존 5개 층의 패턴이 전체에서 유지 → 현재 선택 충분
- 다른 층에서 더 높은 AUC 또는 다른 패턴 → 층 선택이 결과에 영향

---

### A5: 다중 토큰 결정 시점

**소요: 1일 (데이터 재추출 필요)**

| 검증 대상 | V10 선택 | 문제 |
|---------|---------|------|
| 결정 시점 토큰 | 마지막 1개 | 의사결정이 여러 토큰에 걸쳐 형성될 수 있음 |

**방법**: 결정 시점의 마지막 토큰뿐 아니라 이전 2-3개 토큰의 활성화도 추출한다. 세 가지 방식으로 결합하여 분류 AUC를 비교한다.

| 결합 방식 | 설명 |
|---------|------|
| Last only (현재) | 마지막 토큰만 |
| Mean pool | 마지막 3개 토큰 평균 |
| Concat | 마지막 3개 토큰 연결 (150D) |

**판단 기준**:
- AUC 유사 → 마지막 토큰으로 충분
- AUC 상승 → 정보가 분산되어 있음, 현재 과소 추정

**주의**: 기존 게임에서 토큰 위치를 재계산해야 하므로 V10의 데이터 추출 코드 수정이 필요하다.

---

## 실행 순서 요약

```
A1 (PCA 민감도)      2-3시간   ← 가장 빠르고 파이프라인 핵심 검증
A3 (비선형 분류기)    3-4시간   ← A1과 병렬 실행 가능
A2 (순열 1000회)     반나절    ← 계산만 돌려놓으면 됨
A4 (LLaMA 전체 층)   반나절    ← A2와 병렬 가능
A5 (다중 토큰)       1일       ← 데이터 재추출 필요, 마지막
```

A1-A4는 기존 추출 데이터를 그대로 쓰므로 **총 1-2일**이면 전부 가능하다. A5만 데이터 재추출이 필요하다.
